# Decks

A deck is one folder. Drop an HTML file and standardized video files into it. Visit `http://localhost:3000/d/<folder-name>` to present. Share that browser window in Riverside to record.

## Folder convention

```
decks/<your-deck>/
  index.html        ← any .html file works; index.html is preferred
  video-one.mp4
  video-two.mp4
  video-three.mp4
  video-four.mp4
  video-five.mp4
  video-six.mp4
```

Other media types (`.webm`, `.mov`, `.m4v`, `.jpg`, `.png`, `.svg`) work too — anything in the folder is served at `/d/<your-deck>/<filename>`.

## HTML convention

Reference videos by filename, no folder prefix:

```html
<video src="video-one.mp4" autoplay loop muted playsinline></video>
```

The server injects a `<base href="/d/<your-deck>/">` tag so relative paths resolve to the right folder.

Wrap each `<video>` in a `<div class="video-slot">` to get auto-hide behavior — the slot only renders when the file actually loads. The template at `_template/index.html` demonstrates the pattern.

## Workflow

1. **Start a new deck** — copy the template:
   ```
   cp -r decks/_template decks/my-deck
   ```
2. **Author the HTML** — edit `decks/my-deck/index.html` directly, or paste in HTML generated by an LLM (Gemini, Claude, ChatGPT). Give the model this README as context so it uses the right slot names.
3. **Drop in videos** — `video-one.mp4` through `video-six.mp4` as needed. Empty slots stay hidden.
4. **Open it** — `http://localhost:3000/d/my-deck`. The browser auto-reloads when any file in the folder changes.
5. **Present and record** — press `F` for fullscreen. Arrow keys / space / click to advance. In Riverside, share the browser window.

## URL behavior

- `/d` — list of all decks
- `/d/<deck>` — serves the HTML with live reload injected
- `/d/<deck>/<file>` — serves any sibling file (videos, images, css, etc.)
- `/d/<deck>?live=0` — disables auto-reload

## Prompt snippet for LLMs

Paste this when asking an LLM to generate a deck:

> Generate a single self-contained `index.html` for a presentation slide deck. Each slide is a `<section class="slide">`. To embed a video, use `<div class="video-slot"><video src="video-one.mp4" autoplay loop muted playsinline></video></div>`. Use only these video filenames in order: `video-one.mp4`, `video-two.mp4`, ..., `video-six.mp4`. Don't reference any other media files. Use only system fonts. The host environment scales the page to a 16:9 stage and handles keyboard navigation, so you don't need to add scripts for that.
